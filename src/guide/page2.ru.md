# Настройте алгоритм под себя

**1. Слайдеры (0-10):** Выберите приоритеты.
*Дешевое = 10* → Приложение высоко оценивает **бюджетные** блюда.
*Дешёвое = 0* → Критерий отключён.

**2. Инвертируйте приоритет!** Тап по критерию меняет его на противоположный:
*Дешёвое* ↔  *Дорогое*.
Дорогое сменится на Дешевое, и обратно. 
Если выбрано **«Дорогое»**, высокую оценку получат уже *дорогие* блюда.

<img src="./tap_to_invert.webp" alt="Анимация: Нажатие на критерий для инверсии" class="onboarding-animation" />

**3. Экономические зоны:** Меняйте регион, и расчёт стоимости произойдёт по местным ценам.

**4. Пресеты:** Сохраните текущие настройки через кнопку *Сохранить*. Пресеты хранятся локально на вашем устройстве.

<summary><strong>Цены в $/1000ккал?</strong></summary>

Приложение использует *3 варианта оценки стоимости*: `$/кг`, `$/порция` и `$/1000ккал`. Зачем нужны все 3?

Сравнивать цену двух блюд за порцию сложно — порции блюд различаются по размеру.

По весу (кг) тоже не всегда очевидно: например, литр чая стоит копейки, причём он вкусный и быстрый в приготовлении
В пересчёте на кг чай — лучшее «блюдо». Единственный минус — в нём около 0 калорий.

Поэтому в приложении есть все три метрики — выбирайте удобный вариант под вас. (А вот чая в рейтинге нет совсем!)

</details>
